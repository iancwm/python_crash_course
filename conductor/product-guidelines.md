# Product Guidelines

## 1. Prose Style

*   **Informal & Engaging**: Explanations and comments within the notebooks should adopt an informal and engaging tone. The goal is to make learning accessible and enjoyable, using conversational language that resonates with the target audience ("Rick"). Avoid overly formal academic language where a simpler, more direct explanation would suffice.

## 2. Visual Aesthetic

*   **Clean & Minimalist**: All visual elements, including charts, code formatting, and general layout within the notebooks, should prioritize a clean and minimalist aesthetic. This ensures readability and reduces cognitive load, allowing the learner to focus on the core concepts. Visualizations should be clear, concise, and avoid unnecessary clutter.

## 3. Notebook Structure

*   **Interactive & Exploratory**: The Jupyter Notebooks should be structured to encourage active participation and experimentation. This means providing opportunities for the learner to modify code, run cells, and explore different outcomes. Encourage curiosity and self-discovery within a guided framework. Notebooks should be easy to navigate, with clear headings and a logical flow that supports an exploratory learning path.
